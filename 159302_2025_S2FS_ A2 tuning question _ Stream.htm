Hi Reece,

Here are some guidelines for calbirating the fuzzy controller:

1.     INPUTS:

        The start-up codes have been preset to accept two inputs already, representing X and Y, conforming to the requirements of Yamakawas design.  In the start-up codes, inside main.cpp, you will find the statements on line #429 and #430, involving the inputs.

 Code Snippet:


 2.      COEFFICIENTS:

        The variables holding the coefficients A,B,C and D have been defined already, but their values need to be modified inside fuzzylogic.cpp.

 

        It was mentioned in Yamakawas research paper that all these constants are positive valued.  My clue is that they could be fractions as well (e.g., 0.5, etc.)

       The coefficients allow us to define which of the parameters (x, theta, x, theta) is more important than the others, when making a control decision.

       Based on my experience theta is more important (has more weight) than theta_dot, while x is more important than x-dot.

       That means, when setting the coefficients, set the values using the following guide:

o   C > D

o   A > B

Moreover, you may set the constants A, B, C and D so that the range of values for INPUT X and INPUT Y would fall roughly around the range -4.0 to 4.0.  Therefore, the fuzzy sets for X and Y will be around this range as well.  You can perform experiments around this range to get very good results.

To check, you may perform some experiments and print out the maximum and minimum values for X and Y, then check to see if those values are within the range that you have specified.  

Experiments for finding the range of values for INPUT_X and INPUT_Y:

You may perform some experiments to check the range of values:  As shown in the code snippet below, set the initial angle to -45.0 and initial x to 1.0 (see sample statements below), then let the simulation run to see what the minimum and maximum values for INPUT_X and INPUT_Y will be. 

                          prevState.x = 1.0;
                          prevState.angle = -45.0 * (3.14 / 180);
                          prevState.x_dot = 0.0;
                          prevState.angle_dot = 0.0;
                          prevState.angle_double_dot = 0.0;
                          prevState.x_double_dot = 0.0;

 You can write some additional codes (simple if then statements) to find the minimum and maximum values automatically.  Moreover, perform an experiment using an opposite setting.  As shown in the code snippet below, set the initial angle to +45.0 and initial x to -1.0, then let the simulation run to see what the minimum and maximum values for INPUT_X and INPUT_Y will be.

                           prevState.x = -1.0;

                          prevState.angle = 45.0 * (3.14 / 180);
                          prevState.x_dot = 0.0;
                          prevState.angle_dot = 0.0;
                          prevState.angle_double_dot = 0.0;
                          prevState.x_double_dot = 0.0;

 By adjusting the coefficients and performing the experiments, we will be able to set the coefficients A, B, C and D so that the range of values for INPUT_X and INPUT_Y would be roughly within [-4, 4].

 3.      NUMBER OF RULES, NUMBER OF OUTPUTS, NUMBER OF FUZZY SETS

       The number of rules and the number of outputs need to be configured according to your implementation.  Yamakawa defines 13 rules, omitting 12 rules in a 5x5 FAMM (25 rules).  Some students prefer writing the complete 25 rules, as they find it easier to calibrate than a 13 rule FAMM.

         The variable, named no_of_inp_regions correspond to the number of fuzzy sets assigned per input.

        The start-up codes have been designed to use a single FAMM, which conforms to Yamakawas design, but it can also be slightly tweaked to implement 2 FAMMs, if preferred.  Since the required implementation must follow Yamakawas design, the implementation of the rule evaluation and defuzzification steps can be kept as is.

  

4.     FUZZY RULES:

 It is possible to implement 13 rules, as described in Yamakawas journal, or extend the rules to 25 rules, if preferred.  Open fuzzylogic.cpp and place the rules inside

        void initFuzzyRules(fuzzy_system_rec *fl)

 Inside void initFuzzySystem (fuzzy_system_rec *fl):

       Change the number of rules

       Change the number of outputs (if necessary)

       Change the settings for the coefficients.

       Define the rule outputs

 Please note that the fuzzy rules in the start-up code are provided as examples only and should be modified to align with those specified in Yamakawas research paper, as shown in the slide below. Yamakawa uses inputs X and Y (i.e., in_X, in_Y). Additionally, there are 13 rules defined in Yamakawas FAMM, so you will need to incorporate the remaining rules. For more details, refer to the document, titled Week_06_159302_Fuzzy_Logic_Engine_and_Assign_2_stream.pdf.

 

The 13 rules are given in Yamakawas paper, but the actual values assigned to each rule output (e.g. NS, NM, NL, etc. will have to be empirically derived).  I recommend using values that could go more than 100 Newtons for the positive Forces, and less than -100 Newtons for negative forces.   This is due to the physics dynamics of the system the Force required to influence the motion of the cart-pole is dependent on the combination of its weight, angle, angular velocity, horizontal velocity, etc. of the cart-pole system.  Using the recommended range of Forces, your controller will be able to handle bigger initial angles (either Positively Large or Negatively Large angles) for the pole.  This is what we want, to make the controller more robust. 

  

5.      MEMBERSHIP FUNCTIONS:

We can pick-up some useful hints from Yamakawas paper.  Firstly, we can gather from the paper that the membership functions are all triangular, symmetric and centred at zero.  We can also see how the vertices intersect each other.


Figure from Yamakawas paper

To help customise the parameters of your membership functions, use the Excel Fuzzy Set viewer (i.e. Fuzzy_Sets_Viewer.xlsx) I have provided to the class.

Lastly, as a clue, it is possible to handle all possible inputs by using Fuzzy sets at both ends of the spectrum that are opening to the left/opening to the right.  You dont need to copy Yamakawas fuzzy sets that are all triangular fuzzy sets.

 

////////////////////////////////////////////////////////////////////////////////////////////

6.      RULE CALIBRATION

INITIAL CALIBRATION STAGE:

Start with relatively easier initial states during initial stage of calibration

During the initial stage of your calibration, start with relatively easy initial states first (e.g. x=0.0, initial angle = 0.0), then adjust the fuzzy rule outputs (Force), to try and keep the pole balance and centred across the platform.  Work your way to more challenging initial states afterwards.

In the start-up codes, for debugging purposes, look for the following line to set the initial x-coordinate of the cart-pole system.  However, before submission, you need to set it back to 1.0.


To help with the debugging process, print out the value of INPUTS X and Y, then check to see which specific rule must be firing for that particular case.  For example, if you will subdivide the range of values roughly equally according to the fuzzy sets NM, NS, ZR, PS and PM, then you will be able to tell which rule must have a higher degree of firing than the others. 

Example: 

If you have assigned rule outputs within the range -100 to 100 Newtons, and if the rule states PS, then dividing the range of rule outputs equally to define {NM, NS, ZR, PS, PM}, then that rule roughly generates a value of 50 Newtons, in place of PS. 

 

If for a particular X and Y combination of values, you are expecting the rule output to be close to 50 Newtons, check to see if your fuzzy system generates an actual output close to 50 Newtons.  If not, then identify to which rule output your actual output comes close to, you can then adjust the range of values for your fuzzy sets (that means, the parameters for your membership functions).  

Cheers,
Napoleon
